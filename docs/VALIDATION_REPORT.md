# Validation Report

Confirms whether this release repository contains everything needed for
reviewers and researchers to reproduce the implementation and evaluation of
*"A Privacy-Focused, Post-Quantum-Oriented Digital Identity System Using
Blockchain and Zero-Knowledge Proofs"* (Access-2026-15409), and records exactly
what was verified and how.

**Verdict:** ✅ The repository is **complete and reproduction-ready**. Every
artifact needed to reproduce the paper's claims is present (or regenerated by a
committed, pinned script), and the protocol/crypto/circuit/contract/benchmark and
test source is **byte-for-byte identical** to the source prototype that produced
the paper's evaluation.

> **Update — reconciled to submitted manuscript V6.6.** The embedded paper-claim
> reference values in `harness/generate_results.ts` were versioned to the
> submitted revision (V6.6, which adopted the prototype's controlled
> measurements). `npm ci`, `tsc --noEmit`, and `eslint` all pass in this
> repository, `harness/generate_results.ts` was re-run, and the regenerated
> `RESULTS.md` + `MANUSCRIPT_RECONCILIATION.md` now show **0 divergences** — the
> paper and the prototype report the same numbers. The documentation
> discrepancies B1–B3 (REPRODUCE.md zkey hash, `verify-pins` script,
> `vc-schema.json` comment) are fixed. See [`MISSING_ARTIFACTS.md`](MISSING_ARTIFACTS.md).
>
> **CI is green.** GitHub Actions (`.github/workflows/ci.yml`) runs on every push
> to `main`. On a clean `ubuntu-24.04` runner it builds the native toolchain
> (liboqs/GMP/rapidsnark/venv), runs `npm ci` and `npm run setup` (**ptau + zkey
> pins verified**), and passes lint + typecheck, the cross-platform **circuit +
> determinism** vitest suites, the **dilithium-py↔liboqs interop**, and the
> **revoked-credential negative test**. The Linux `test` job is scoped to those
> platform-agnostic suites because the `protocol`/`kyber`/`ledger_verifier`/
> `bench_env` suites and the timed benches exercise the Windows+WSL2 host (PQC
> bridge via `wsl.exe`; §B guard via `powershell`) and are validated there. The
> green run is itself a fresh-checkout clean-environment validation of the
> published commit.

## 1. Method of validation

This release was produced by **copying** the source prototype (never moving or
altering it) and adding documentation. Validation was performed as follows:

| Check | Method | Result |
|---|---|---|
| Functional/scientific identity | SHA-256 of **every** copied file vs. its source counterpart | **141/141 identical, 0 mismatches** |
| File-count parity | `find` count, source (filtered) vs. dest | 141 = 141 |
| Inclusions present | existence check of circuit build, zkey, vkey, verifier, pins, results, sample, fixtures, lockfile | all present |
| Exclusions correct | existence check of `node_modules`, `.git`, binaries, research r1cs | all absent |
| Pinned artifact integrity | SHA-256 of committed zkey/vkey/ptau vs. `setup/pins.json` | zkey `63529c2b…` ✓, vkey `79d31eb1…` ✓, ptau `3ef2ecc5…` ✓ |
| No broken/absolute paths in code | repo-wide search for `C:\…` and `/pqid-prototype/` path segments | only in `circuits/build/circuit_info.json` `circomStdout` (build-log provenance, not a code path) |
| Workspace module resolution | export maps vs. imports for all `@pqid/*` packages | consistent (`@pqid/common/*`, `@pqid/pqc`, `@pqid/wallet/*`, `@pqid/ledger/*`, `@pqid/harness/e2e-core`, etc.) |
| Relocation safety | `REPO_ROOT` derivation in `packages/common/src/paths.ts` | computed from `import.meta.url` (`../../..`); independent of the repo folder name |

> **What was executed here.** `npm ci` (clean lockfile install), `tsc -p
> tsconfig.json --noEmit` (typecheck — exit 0), `eslint` on the edited generator
> (exit 0), and `harness/generate_results.ts` (regenerated `RESULTS.md` +
> `MANUSCRIPT_RECONCILIATION.md`, 0 divergences). This confirms the public copy
> installs, typechecks, lints, and runs its reporting pipeline in a from-scratch
> dependency install.
>
> **What was NOT executed here, and why.** The native-dependent test suites
> (`protocol`, `kyber`, `ledger_verifier`, interop) and the timed benchmark
> campaign were **not run here**: they require the native Linux toolchain
> (liboqs 0.15.0, rapidsnark v0.0.8, GMP 6.3.0), Foundry's `anvil`, and
> PostgreSQL, plus — for the timed campaign — a quiesced machine for ~1 hour
> under the guard-protected §B discipline. Re-running them is the reviewer's
> reproduction step (documented in `SETUP.md`/`REPRODUCE.md`); `.github/workflows/ci.yml`
> is the clean-host recipe that exercises them and is run on the public repo.
> Functional equivalence to the source is established by **byte-identical hashing
> of every protocol/crypto/circuit/contract/benchmark/test source file** (see §5),
> so any behaviour the source prototype exhibited (and that produced the committed
> `results/*.json`) the copy reproduces identically.

## 3. Reproducibility risks

| Risk | Severity | Mitigation in this repo |
|---|---|---|
| Hardware-dependent timings differ from the paper | Expected, low | Every result embeds host/CPU/OS/toolchain; `RESULTS.md` auto-generates a divergence table; the scientific claims are hardware-robust |
| Thermal state changes ULV-laptop proving times | Documented | Both cool and sustained states are measured and reported (`zk_rapidsnark_cool.json`, `RESULTS.md` thermal table) |
| Native toolchain build is involved | Medium | `docker/` images + `docker/provision-wsl.sh` + CI workflow give three reproducible recipes |
| External ptau download URL could rot | Mitigated | The pinned ptau is **committed** in this release (`setup/ptau/…`), so setup does not depend on the network |
| Platform binaries (circom/Foundry) not committed | Low | `tools/README.md` gives pinned download sources; Linux/CI use `PATH`; Docker uses the official Foundry image |
| `npm run verify-pins` doc/script mismatch | Resolved | `verify-pins` npm script added (`MISSING_ARTIFACTS.md` B2) |
| Stale zkey hash strings in `REPRODUCE.md` | Resolved | Corrected to `63529c2b…`; pin/artifact/manuscript (V6.6) agree (`MISSING_ARTIFACTS.md` B1) |
| RESULTS/reconciliation showed old "Paper" numbers | Resolved | Paper-claim constants versioned to V6.6; regenerated → 0 divergences |
| Trusted setup is a dev beacon, not an MPC ceremony | By design `[A]` | Stated in `setup/groth16_setup.ts`, `REPRODUCE.md`, and the paper |

No risk in this table blocks reproduction of a measured claim.
